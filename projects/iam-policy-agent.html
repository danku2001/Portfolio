Problem / Goal

AWS IAM policies can contain risky permissions such as wildcards, public principals, dangerous actions, or privilege escalation paths. The goal was to build an AI-assisted review tool that detects these risks and suggests safer alternatives.

Key Features

  • Detects wildcard permissions and overly broad access patterns.
  • Identifies public or external principals in IAM policies.
  • Flags dangerous IAM actions and risky permission combinations.
  • Analyzes potential privilege escalation risks.
  • Generates structured security reports with risk verdicts, findings, and remediation steps.
  • Suggests safer policy fixes and clearer security recommendations.

Technical Highlights

  • Designed a modular multi-agent architecture for security analysis.
  • Implemented an action-classification agent for IAM permission understanding.
  • Built a main orchestration agent that coordinates tools and analysis workflow.
  • Added an independent verifier agent to review outputs and improve reliability.
  • Integrated Groq-hosted Llama models with function-calling workflows and automated tool orchestration.

What I Learned

This project strengthened my understanding of LLM agents, function calling, cloud security, AWS IAM risk patterns, structured output design, and how to build AI systems that support practical security review workflows.
