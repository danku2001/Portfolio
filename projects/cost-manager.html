Cost Manager API

A backend expense-management system for tracking user expenses and generating monthly reports.

Node.js Express.js MongoDB Mongoose REST API Jest Supertest
Back to Projects GitHub

Problem / Goal

The goal of this project was to build a structured backend system that manages users, expenses, logs, and monthly cost reports through REST APIs.

Key Features

  • User management through a dedicated users service.
  • Expense creation and categorization through a costs service.
  • Monthly reports by user, year, and month.
  • Request validation for user IDs, amounts, categories, and dates.
  • Logging system for service requests.
  • Separate services for users, costs, logs, and about/team information.
  • Automated API tests using Jest and Supertest.

Technical Highlights

  • Built REST API routes with Express.js.
  • Connected the backend to MongoDB using Mongoose.
  • Separated responsibilities across multiple services.
  • Used validation and logging to make the API easier to debug and maintain.
  • Tested endpoints with automated API tests.

What I Learned

This project helped me practice backend service separation, REST API design, database modeling, validation, logging, and automated testing.
