Store Network System

A Java client-server system that simulates a multi-branch clothing store network.

Java OOP Sockets Threads JSON Design Patterns

Quick facts

Project type Java system project
Main stack Java, sockets, threads, JSON
Focus Client-server architecture
Repository Public GitHub project

Problem / Goal

The goal was to model a larger business system with multiple branches, employees, customers, inventory, purchases, permissions, reports, and communication between branches.

Key Features

  • Server that handles multiple clients using threads.
  • Console-based client application.
  • User authentication and role-based permissions.
  • Duplicate login prevention.
  • Inventory, employee, customer, and purchase management.
  • Reports by branch, product, and category.
  • Inter-branch chat and communication.

Technical Highlights

  • Used Java sockets for client-server communication.
  • Applied OOP principles such as inheritance, polymorphism, and interfaces.
  • Used design patterns such as Singleton, Strategy, and Observer.
  • Handled a larger system structure with multiple responsibilities and entities.

What I Learned

This project helped me practice Java system design, object-oriented programming, sockets, multithreading, role-based flows, and organizing a larger codebase.
