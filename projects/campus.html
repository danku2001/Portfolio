Problem / Goal

The project simulates a smart-campus system where environmental data such as temperature and light can be monitored across campus areas, and shading behavior can be controlled manually or automatically.

Key Features

  • React frontend for managing areas, sensors, and shading controls.
  • Node.js and Express backend with REST API endpoints.
  • MySQL database for persistent data storage.
  • Sensor management features for adding, editing, and deleting sensors.
  • Automatic shading logic based on environmental readings.
  • Manual override for administrators.
  • Responsive dark-themed interface.

Technical Highlights

  • Built a full-stack flow from UI to API to database.
  • Modeled realistic smart-campus / IoT-style business logic.
  • Separated frontend and backend responsibilities.
  • Used REST APIs to connect the React client with the backend.

What I Learned

This project strengthened my full-stack development skills, especially React frontend structure, Express APIs, MySQL data modeling, and connecting user actions to backend logic.
