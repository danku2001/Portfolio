Problem / Goal

The project implements a two-player Connect Four game where one player creates a room and another joins using a code. The goal was to practice client-server thinking, game state, and turn-based logic.

Key Features

  • Room creation with a game code.
  • Two-player joining flow.
  • Turn management.
  • Server-side game-state handling.
  • Win and draw detection.
  • Client, server, and database separation.

Technical Highlights

  • Implemented core game logic in JavaScript.
  • Used Node.js for backend game flow.
  • Used MySQL for persistent game-related data.
  • Practiced synchronization between client actions and server state.

What I Learned

This project helped me understand how to structure interactive game logic, manage state, and connect a simple frontend to backend and database layers.
